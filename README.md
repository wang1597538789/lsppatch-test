# LSPatch 实践：阻止目标 App 到期自动卸载

## 📖 项目简介
本项目是一个基于 [LSPatch](https://github.com/LSPosed/LSPatch) 的免 Root Hook 实践项目。主要目的是通过注入自定义模块，阻止特定的目标 App 在授权或试用期结束后执行自动卸载操作。

## ⚙️ 工作原理
本模块主要通过 Hook 以下两个系统层面的关键点来实现功能：
1. **Hook 系统时间**：修改目标 App 获取到的系统时间，使其认为当前仍处于有效期内。
2. **拦截卸载命令**：Hook 并阻断目标 App 内部调用系统卸载（Uninstall）相关 API 或命令的过程，防止其在后台“自尽”。

## 🛠️ 使用方法

要将此模块的补丁打入目标 App，你需要使用 LSPatch 提供的命令行工具进行打包。

### 准备工作
* 准备需要被处理的 **目标 App** 安装包（例如：`target-app.apk`）。
* 编译本项目生成 **模块 App** 安装包（例如：`module-app.apk`）。
* 下载 LSPatch 仓库发布页的 jar 包（本项目使用的版本参考：`jar-v0.6-398-release.jar`）。

### 执行打包
在命令行或终端中运行以下命令，将模块合并到目标 App 中：

```bash
java -jar jar-v0.6-398-release.jar [目标app.apk路径] -m [模块app.apk路径]


执行成功后，会在当前目录下生成一个注入了模块补丁的全新 APK 文件。

⚠️ 注意事项与局限性
重新签名问题：通过 LSPatch 重新打包后，生成的新 App 会被重新签名，其签名信息与官方原版不一致。

签名校验导致的功能失效：如果目标 App 的某些核心功能（或系统权限）强依赖于原版的签名校验，或者 App 内部包含严格的完整性/签名校验机制，重新打包后的 App 可能会出现功能异常、闪退或无法安装的情况。
